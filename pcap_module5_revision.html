Παγίδες & Προσοχή

Οι λεπτομέρειες που τρώνε πόντους από οριακούς μαθητές. Αυτές χωρίζουν το «το ξέρω περίπου» από το «το πετυχαίνω σίγουρα».

📌 Conditional expression: τιμή-αν-αληθές ΠΡΩΤΑ

Η σύνταξη είναι a if cond else b — δηλαδή η τιμή για το αληθές μπαίνει πρώτη, μετά το if, μετά το else. Π.χ. [1 if x%2==0 else 0 for x in range(5)] → [1,0,1,0,1].

📌 Σύνταξη lambda — τι ΔΕΝ επιτρέπεται

lambda x: x+1 ✓ και lambda: 2 ✓ (χωρίς παραμέτρους). Αλλά: όχι return μέσα (lambda x: return x+1 → SyntaxError), όχι παρενθέσεις γύρω από τις παραμέτρους, όχι = στη θέση του :.

📌 map & filter επιστρέφουν iterator

Δεν επιστρέφουν λίστα! Για να δεις τις τιμές, τύλιξέ τα σε list(). Επίσης διατρέχονται μία φορά — μετά «αδειάζουν».

📌 filter: κρατά τα True

Το filter(fn, seq) κρατά τα στοιχεία για τα οποία η fn επιστρέφει True και απορρίπτει τα υπόλοιπα. Π.χ. filter(lambda x: x>0, [-1,0,1,2]) → [1, 2].

📌 Το generator function δεν επιστρέφει τις τιμές

Η κλήση μιας συνάρτησης με yield επιστρέφει ένα generator object, όχι τη σειρά τιμών. Για τις τιμές: list(gen()) ή βρόχος for.

📌 Iterator protocol = __iter__ + __next__

Ένας iterator υλοποιεί δύο μεθόδους: __iter__() (επιστρέφει τον εαυτό του) και __next__() (επόμενη τιμή· πετάει StopIteration στο τέλος).

📌 Closures «θυμούνται» το περιβάλλον

Μια εσωτερική συνάρτηση κρατά πρόσβαση στις μεταβλητές της εξωτερικής ακόμη κι αφού η εξωτερική έχει επιστρέψει. Αυτό είναι το closure.

📌 Το 2ο όρισμα του open() είναι string

Το mode είναι string (π.χ. "r", "wt", "rb") — όχι ακέραιος. Λάθος τύπος → TypeError.

📌 Τι κάνει κάθε mode (κλασική παγίδα)

'r' → πρέπει να υπάρχει. 'w' → μηδενίζει/δημιουργεί. 'a' → append. 'r+' → read/write αλλά πρέπει να υπάρχει. 'w+' → read/write αλλά μηδενίζει. Το 'b' κάνει binary, το 't' (default) text.

📌 open() δεν επιστρέφει False σε αποτυχία

Επιστρέφει file object· αν αποτύχει (π.χ. ανύπαρκτο αρχείο σε mode "r"), πετάει εξαίρεση (FileNotFoundError / IOError), δεν επιστρέφει False.

📌 Standard streams & close()

Τα stdin, stdout, stderr είναι προ-ανοιγμένα streams· το print() γράφει στο stdout. Για να κλείσεις τη σύνδεση handle–αρχείου χρησιμοποιείς .close().
