Παγίδες & Προσοχή

Οι λεπτομέρειες που τρώνε πόντους από οριακούς μαθητές. Εδώ, στο 34% του exam, παίζεται η επιτυχία.

📌 Το self είναι υποχρεωτικό στις μεθόδους

Κάθε μέθοδος (και το __init__) έχει τουλάχιστον την παράμετρο self. Αν την παραλείψεις (def __init__():) και δημιουργήσεις αντικείμενο → TypeError.

📌 Class.method(obj) vs obj.method()

Με obj.method() το self περνά αυτόματα. Αν καλέσεις από την class (Storage.get(stuff)), πρέπει να δώσεις ρητά το αντικείμενο ως πρώτο όρισμα.

📌 Ανάθεση μέσω instance δημιουργεί instance variable

Η a.v = 5 δεν αλλάζει το class variable — φτιάχνει νέο instance variable που «σκιάζει» το class. Τα άλλα instances και το A.v μένουν ανέπαφα.

Π.χ. μετά από a.v = 5: a.v → 5 αλλά b.v → 1 και A.v → 1.

📌 Class variable = κοινό για όλους

Αν τροποποιήσεις το class variable μέσω της class (π.χ. Package.spam += ...), η αλλαγή φαίνεται σε όλα τα instances — μοιράζονται την ίδια τιμή.

📌 Name mangling: __x → _Class__x

Ένα όνομα με δύο leading underscores «μπερδεύεται» (mangling). Πρόσβαση απ' έξω μόνο ως obj._ClassName__x ή ClassName._ClassName__x. Σκέτο obj.__x ή obj.x → AttributeError.

📌 issubclass(B, B) είναι True

Κάθε class θεωρείται υποκλάση του εαυτού της. Προσοχή στη σειρά ορισμάτων: issubclass(parent, child) → False, issubclass(child, parent) → True.

📌 isinstance: γονέας ΔΕΝ είναι instance του παιδιού

Αν το o είναι instance του superclass, τότε isinstance(o, Subclass) → False. Το αντικείμενο δεν μπορεί να είναι instance μιας πιο εξειδικευμένης class.

📌 Αναζήτηση ιδιότητας: instance → class → superclasses

Αν δεν βρεθεί στο instance, η Python ψάχνει στην class και μετά στους superclasses (κατά MRO). Γι' αυτό ένα Bottom() «βλέπει» το value που ορίστηκε στον Middle.

📌 MRO: αριστερά → δεξιά, κάτω → πάνω

Σε class C(BR, BL), αν και οι δύο έχουν do(), κερδίζει το αριστερό (BR). Η αναζήτηση ξεκινά από το αντικείμενο και ανεβαίνει.

📌 Παράνομο MRO → TypeError

Ένας superclass δεν μπορεί να προηγείται μιας υποκλάσης του στη λίστα βάσεων. Π.χ. με Left(Top), το class Bottom(Top, Left) → TypeError, ενώ class Bottom(Left, Top) είναι νόμιμο.

📌 super() & σιωπηρός constructor

Αν μια subclass δεν ορίζει δικό της __init__, καλείται αυτόματα ο constructor του superclass. Αν ορίζει, ο superclass constructor καλείται μόνο αν τον καλέσεις ρητά (super().__init__()).

📌 String αναπαράσταση → __str__

Για να εμφανίζεται «ωραία» ένα αντικείμενο με print(), ορίζεις __str__() — όχι __string__, ούτε string().

📌 is (ταυτότητα) ≠ == (τιμή)

Το is ελέγχει αν δύο μεταβλητές δείχνουν στο ίδιο αντικείμενο. o2 = o1 → o1 is o2 True· δύο ξεχωριστά A() → False.
